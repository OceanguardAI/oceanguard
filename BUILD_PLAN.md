## What We Are Building

**Problem:** IUU fishing costs 11–26M tonnes of fish per year. Conservation officers watching Marine Protected Areas (MPAs) cannot manually cross-reference satellite radar imagery against AIS vessel broadcasts at scale. "Dark" vessels disable their AIS — they're invisible to tracking systems but visible to satellite SAR.

**OceanGuard:** Automates that cross-reference. SAR detection → AIS check → MPA proximity → deterministic risk score → Claude AI explanation → human officer decides.

**Demo MPA:** Bar Reef Marine Sanctuary, Sri Lanka.

**Architecture principle:** Deterministic, auditable scoring (never a black box) + LLM narration (explains the score). The algorithm decides the risk level; Claude explains it.

## Risk Scoring Engine (exact formula — implement verbatim)

```python
def calculate_risk(detection_conf: float,
                   ais_matched: bool,
                   ais_data_available: bool,
                   inside_mpa: bool,
                   near_mpa: bool,
                   image_quality_score: float,  # 0..1
                   fishing_score: float = 0.0,
                   repeated_activity_score: float = 0.0):

    effective_conf = detection_conf * image_quality_score

    if not ais_data_available:
        ais_score = 0.3        # neutral — absence of data ≠ guilt
    else:
        ais_score = 0.0 if ais_matched else 1.0

    mpa_score = 1.0 if inside_mpa else (0.6 if near_mpa else 0.0)

    risk = (0.30 * effective_conf +
            0.25 * ais_score +
            0.25 * mpa_score +
            0.10 * fishing_score +
            0.10 * repeated_activity_score)

    if   risk >= 0.75: level = "CRITICAL"
    elif risk >= 0.55: level = "HIGH"
    elif risk >= 0.35: level = "MEDIUM"
    else:              level = "LOW"

    return round(risk, 3), level
```

**AIS matching rule:** spatial ≤2 km + time window ±3 hours.
**Near-MPA threshold:** ≤5 km.
**Confidence threshold:** 0.45 (tuned for recall — missing a dark vessel is worse than a false alarm).

**Worked example (bar-reef-003):**
- conf=0.70, ais_matched=False, ais_data_available=True, inside_mpa=False, near_mpa=True, image_quality=1.0
- `0.30*0.70 + 0.25*1.0 + 0.25*0.6 + 0 + 0 = 0.21 + 0.25 + 0.15 = 0.61 → HIGH`

## Agent Prompts & Fallbacks

### narrator — `POST /agents/narrate`
**System:** "You are a marine conservation analyst providing decision-support to patrol officers. Explain detections clearly. Never make accusations or assign guilt. Always note what is uncertain. The officer makes all final decisions."

**User:** "A vessel has been detected at {lat:.4f}°N, {lon:.4f}°E with SAR confidence {sar_confidence:.0%}. AIS data was available but no matching broadcast was found within 2 km / 3 hours ({matching_method}). The vessel is {distance_to_mpa_km:.1f} km from {mpa_name} (near_mpa={near_mpa}). Risk score: {risk_score} ({risk_level}). In 2-3 sentences, explain why this was flagged and what makes this uncertain. Return JSON: {\"why_flagged\": \"...\", \"uncertainty\": \"...\"}"

**Fallback (no API key):**
```python
why_flagged = (
    f"Vessel detected at {e.lat:.3f}°N, {e.lon:.3f}°E shows no matching AIS broadcast "
    f"within 2 km / 3 hours, placing it {e.distance_to_mpa_km:.1f} km from {e.mpa_name}. "
    f"Risk score {e.risk_score:.2f} ({e.risk_level}) driven by unmatched AIS and MPA proximity."
)
uncertainty = (
    f"SAR detection confidence is {e.sar_confidence:.0%}. "
    "AIS absence may reflect technical failure rather than intent to evade. "
    "A conservation officer should verify against additional imagery and context."
)
```

### briefing — `POST /agents/briefing`
**System:** Same conservation analyst system prompt.
**User:** "Summarise the current situation at Bar Reef Marine Sanctuary. {n} vessel detections have been flagged. Highest risk: {top.id} at {top.risk_score} ({top.risk_level}), {top.distance_to_mpa_km:.1f} km from the MPA. Provide a 3-5 sentence situational briefing and recommend an alertness level (NORMAL / ELEVATED / HIGH)."

**Fallback:** Template string from highest-risk event fields.

### patrol — `POST /agents/patrol`
**System:** Same.
**User:** List all events with fields → "Rank these detections for patrol priority, weighting MPA proximity and unmatched AIS highest. Return JSON array: [{id, rank, justification}]"

**Fallback:** Deterministic sort: inside_mpa first, then near_mpa, then risk_score desc. Template justifications.

### ask — `POST /agents/ask` (tool-calling)
Tools:
```python
tools = [
    {"name": "query_detections", "description": "Filter risk events by source and/or risk level",
     "input_schema": {"type": "object", "properties": {
         "source": {"type": "string", "enum": ["GFW", "YOLO_SAR"]},
         "risk_level": {"type": "string", "enum": ["LOW", "MEDIUM", "HIGH", "CRITICAL"]}
     }}},
    {"name": "get_event", "description": "Get a specific risk event by ID",
     "input_schema": {"type": "object", "properties": {
         "id": {"type": "string"}
     }, "required": ["id"]}},
    {"name": "mpa_distance", "description": "Compute distance from a lat/lon to Bar Reef MPA in km",
     "input_schema": {"type": "object", "properties": {
         "lat": {"type": "number"}, "lon": {"type": "number"}
     }, "required": ["lat", "lon"]}}
]
```
Agentic loop: send message → if tool_use → execute tool → append result → send again → extract final text.

## Two-Proof Narrative (keep these distinct in the UI)

**Proof A — Technology validation (Model Metrics page):**
Custom YOLO11n trained on HRSID (mAP50=0.838), run on a real Sentinel-1 SAR scene from xView3 (scene `590dd08f71056cacv`, Gulf of Guinea). Produced 122 vessel detections, confidences up to 0.76, fully georeferenced. Proves the detector generalises to raw real-world SAR imagery.

**Proof B — Decision-support for Bar Reef (Map Dashboard):**
Global Fishing Watch's SAR-derived dark-vessel dataset gives 4 real unmatched detections near Bar Reef Marine Sanctuary — including one 0.4 km from the boundary. Combined with WDPA MPA polygon + OSM port → risk score → evidence card. This is the headline demo.

## Tech Stack

| Layer | Choice | Why |
|---|---|---|
| Detection model | YOLO11n | Best small-object SAR accuracy per param; 5.5 MB runs on CPU |
| Geospatial | rasterio + pyproj + shapely | Industry-standard raster I/O, CRS transforms, point-in-polygon |
| Backend | FastAPI | Async, typed, auto OpenAPI docs |
| Schemas | Pydantic v2 | One definition, validated at boundary |
| Agents | Anthropic Python SDK (claude-opus-4-8) | Narration, briefing, patrol, Q&A |
| Store (MVP) | JSON file in memory | Right-sized for hundreds of rows |
| Frontend | React + Vite + TypeScript | Fast dev loop, typed |
| Map | react-leaflet | No API key, renders GeoJSON natively |
| Charts | recharts | Declarative metrics charts |
| Icons | lucide-react | Clean maritime/alert iconography |
| Styling | Tailwind | Deep-ocean palette, risk-color tokens |
| Deploy | Docker + docker-compose | One-command run; Cloud Run + Vercel |
