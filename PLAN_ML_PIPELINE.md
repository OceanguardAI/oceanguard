## File 1: `ml/pipeline/tiling.py`

Cuts a large SAR GeoTIFF into 640×640 tiles suitable for YOLO inference.

```python
def tile_sar(
    tif_path: str,
    output_dir: str,
    tile_size: int = 640,
    db_min: float = -50.0,
    db_max: float = 0.0,
    nodata: float = -32768,
    min_valid_frac: float = 0.50,
) -> list[tuple[int, int, str]]:
    """Return list of (row_off, col_off, tile_path)."""
```

**Rules:**
- Use `rasterio` windowed read — never load the full 28676×24522 image into RAM
- Normalise dB values: `uint8 = np.clip((val - db_min) / (db_max - db_min) * 255, 0, 255).astype(np.uint8)`
- Replace `nodata` (-32768) with 0 before normalising
- Skip tile if `valid_pixels / total_pixels < min_valid_frac`
- Write tile as `tile_r{row:04d}_c{col:04d}.png` using Pillow
- Expected result on xView3 scene: **1174 tiles written, 498 skipped**

**Smoke test (`if __name__ == "__main__"`):** tile one small window, print count.

## File 2: `ml/pipeline/detect.py`

Runs YOLO11n inference over tiles. Chunked + resumable to avoid kernel crashes.

```python
def detect_tiles(
    tile_dir: str,
    model_path: str,
    conf_threshold: float = 0.45,
    chunk_size: int = 25,
    checkpoint_path: str | None = None,
) -> list[dict]:
    """Return list of detection dicts."""
```

**Rules:**
- Load model once: `model = YOLO(model_path)`
- Cap threads: `torch.set_num_threads(2)` before inference
- Process in chunks of 25 tiles
- **CRITICAL:** pair results via `zip(batch_paths, results)` — NEVER use `r.path` (unreliable on CPU batch predict)
- Resume: if `checkpoint_path` exists and is a valid JSON, skip tiles already in checkpoint
- Save checkpoint JSON after every chunk (in case of crash)
- Each detection dict:
  ```python
  {
      "tile_path": str,
      "row_off": int,        # from tile filename
      "col_off": int,        # from tile filename
      "x_center_px": float,  # within-tile pixel coords
      "y_center_px": float,
      "width_px": float,
      "height_px": float,
      "confidence": float,
      "class_id": int
  }
  ```
- Parse `row_off` and `col_off` from tile filename: `tile_r{row:04d}_c{col:04d}.png`

**Expected result on xView3 scene:** **122 detections**

## File 3: `ml/pipeline/georeference.py`

Converts pixel coordinates to real-world WGS84 lat/lon.

```python
def georeference_detections(
    detections: list[dict],
    geo_transform_x: float = 477060.79,
    geo_transform_y: float = 793583.94,
    pixel_size: float = 10.0,
    src_crs: str = "EPSG:32631",
    dst_crs: str = "EPSG:4326",
) -> list[dict]:
    """Add 'lat' and 'lon' to each detection dict. Return updated list."""
```

**Formula (exact — do not change):**
```python
from pyproj import Transformer

transformer = Transformer.from_crs(src_crs, dst_crs, always_xy=True)

for det in detections:
    utm_x = geo_transform_x + (det["col_off"] + det["x_center_px"]) * pixel_size
    utm_y = geo_transform_y - (det["row_off"] + det["y_center_px"]) * pixel_size
    lon, lat = transformer.transform(utm_x, utm_y)
    det["lon"] = round(lon, 6)
    det["lat"] = round(lat, 6)
```

Scene details (xView3 `590dd08f71056cacv`):
- CRS: EPSG:32631 (UTM zone 31N)
- Pixel size: 10 m
- Geo-transform origin: x = 477060.79, y = 793583.94
- Location: Gulf of Guinea, West Africa

## File 4: `ml/pipeline/enrich.py`

Computes MPA distance and port distance for each detection.

```python
import json
import math
from shapely.geometry import shape, Point
from shapely.ops import nearest_points


def load_mpa(geojson_path: str):
    """Load Bar Reef MPA polygon from GeoJSON. Returns shapely Polygon."""
    with open(geojson_path) as f:
        data = json.load(f)
    # GeoJSON feature or feature collection
    feature = data if data["type"] == "Feature" else data["features"][0]
    return shape(feature["geometry"])


def distance_to_mpa(lat: float, lon: float, mpa_polygon) -> float:
    """Geodesic distance in km from point to nearest MPA boundary."""
    point = Point(lon, lat)
    if mpa_polygon.contains(point):
        return 0.0
    nearest = nearest_points(point, mpa_polygon.boundary)[1]
    return _haversine(lat, lon, nearest.y, nearest.x)


def classify_mpa(distance_km: float) -> tuple[bool, bool]:
    """Return (inside_mpa, near_mpa). near_mpa = distance_km <= 5.0."""
    inside = distance_km == 0.0
    near = distance_km <= 5.0 and not inside
    return inside, near


def nearest_port_distance(lat: float, lon: float, ports_json_path: str) -> tuple[float, str]:
    """Return (distance_km, port_name) for the nearest OSM port."""
    with open(ports_json_path) as f:
        data = json.load(f)
    # Overpass JSON: data["elements"] list of nodes with lat/lon/tags
    best_dist = float("inf")
    best_name = "Unknown"
    for el in data.get("elements", []):
        d = _haversine(lat, lon, el["lat"], el["lon"])
        if d < best_dist:
            best_dist = d
            best_name = el.get("tags", {}).get("name", el.get("tags", {}).get("leisure", "Port"))
    return round(best_dist, 1), best_name


def _haversine(lat1, lon1, lat2, lon2) -> float:
    R = 6371.0
    dlat = math.radians(lat2 - lat1)
    dlon = math.radians(lon2 - lon1)
    a = math.sin(dlat/2)**2 + math.cos(math.radians(lat1)) * math.cos(math.radians(lat2)) * math.sin(dlon/2)**2
    return R * 2 * math.asin(math.sqrt(a))
```

## Key Gotchas

1. Never load the full SAR GeoTIFF into RAM — use windowed rasterio reads
2. Always use `zip(batch_paths, results)` when pairing YOLO results — `r.path` is unreliable on CPU
3. Chunk inference at 25 tiles + save checkpoint after each chunk — prevents kernel crashes
4. AIS absence (`ais_data_available=False`) gets `ais_score=0.3` (neutral) not 1.0 (guilty)
5. The risk formula weights are fixed — do not tune them without updating docs/data-dictionary.md
